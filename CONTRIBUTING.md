# Contributing to MotmaenBash Android
=====================================

Thank you for considering contributing to MotmaenBash Android. We appreciate your interest in improving our project. This guide outlines the steps to contribute to our repository.

## Forking the Repository
------------------------

To start contributing, fork the `motmaenbash-android` repository:

1. Go to the [motmaenbash-android](https://github.com/motmaenbash-android) repository on GitHub.
2. Click the **Fork** button in the top-right corner of the page.
3. Choose your GitHub account as the destination for the fork.

## Branch Naming Convention
-------------------------

When creating a new branch for your contribution, use the following naming convention:

* `feature/<feature-name>`: For new feature implementations.
* `fix/<issue-number>`: For bug fixes related to a specific issue.
* `chore/<change-description>`: For miscellaneous changes, such as code refactoring or documentation updates.
* `docs/<documentation-change>`: For changes to the project's documentation.

## Commit Messages
-----------------

We follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) specification for commit messages. Ensure your commit messages adhere to the following format:

`type(scope): brief description`

* `type`: One of `feat`, `fix`, `chore`, `docs`, `style`, `refactor`, `perf`, `test`, or `build`.
* `scope`: Optional scope of the commit (e.g., a specific component or feature).
* `brief description`: A brief summary of the changes made.

### Commit Message Example

`feat(login-screen): add forgot password functionality`

## Pull Request Checklist
-------------------------

Before submitting a pull request, ensure you have:

* [ ] Forked the `motmaenbash-android` repository.
* [ ] Created a new branch with a descriptive name.
* [ ] Written clear and concise commit messages following the Conventional Commits format.
* [ ] Included relevant tests for your changes (if applicable).
* [ ] Updated the documentation (if applicable).
* [ ] Formatted your code according to the project's coding standards.
* [ ] Verified that your changes do not introduce any regressions or bugs.

## Submitting a Pull Request
-----------------------------

Once you have completed the above steps, submit a pull request to the `motmaenbash-android` repository. Our maintainers will review your changes and provide feedback.

Thank you for your contribution to MotmaenBash Android!
